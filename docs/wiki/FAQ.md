## 一般的な質問

### Q1: Kuma-Reportは無料ですか？

**A:** はい、Kuma-Reportは完全に無料のオープンソースプラグインです。

---

### Q2: どのMinecraftバージョンに対応していますか？

**A:** Minecraft 1.21以降に対応しています。Paper、Spigot、Purpurサーバーで動作します。

---

### Q3: Bedrock版（統合版）プレイヤーも使用できますか？

**A:** はい。GeyserとFloodgateプラグインを導入していれば、Bedrock版プレイヤーも通報システムを利用できます。Java版とBedrock版で異なるUIが自動的に表示されます。

---

### Q4: 商用サーバーで使用できますか？

**A:** はい、商用・非商用問わず使用できます。MITライセンスの下で公開されています。

---

### Q5: 日本語以外の言語に対応していますか？

**A:** デフォルトでは日本語のみですが、`messages.yml` を編集することで任意の言語に変更できます。英語版のメッセージファイルは今後提供予定です。

---

## 機能に関する質問

### Q6: オフラインのプレイヤーを通報できますか？

**A:** GUIからは通報できませんが、コマンド入力では過去にログインしたことがあるプレイヤーのUUIDが記録されていれば可能です。ただし、現在のバージョンではオンラインプレイヤーのみが対象です。

---

### Q7: 通報したことは相手にバレますか？

**A:** いいえ。通報者の情報は被通報者には表示されません。管理者のみが確認できます。

---

### Q8: 誤って通報してしまいました。取り消せますか？

**A:** プレイヤー自身が通報を取り消すことはできません。管理者に連絡して、通報を却下してもらってください。

---

### Q9: クールダウン時間を変更できますか？

**A:** はい。`config.yml` の `cooldown` セクションで変更できます。

```yaml
cooldown:
  player-report: 60   # プレイヤー通報（秒）
  bug-report: 120     # バグレポート（秒）
```

---

### Q10: 通報の理由カテゴリを追加・削除できますか？

**A:** はい。`config.yml` の `report-categories` セクションで自由にカスタマイズできます。

```yaml
report-categories:
  - name: "カスタムカテゴリ"
    material: DIAMOND
    description:
      - "&#94A3B8カスタムな理由"
```

---

### Q11: 複数のサーバーで通報データを共有できますか？

**A:** 現在のバージョンではSQLiteを使用しているため、同じデータベースファイルを共有することで可能ですが、同時書き込みに弱いため推奨しません。将来のバージョンでMySQLサポートを予定しています。

---

### Q12: 通報データはどこに保存されますか？

**A:** `plugins/Kuma-Report/reports.db` というSQLiteデータベースファイルに保存されます。

---

### Q13: 通報を削除できますか？

**A:** 現在のバージョンでは、通報を削除する機能はありません。承認・却下のみが可能です。データベースファイルを直接編集することで削除できますが、推奨しません。

---

### Q14: スタッフ全員に通知を送りたくない場合は？

**A:** `kumareport.notify` 権限を持たないプレイヤーには通知が送られません。個別に権限を調整してください。

---

### Q15: Discord以外の通知方法はありますか？

**A:** 現在はDiscord Webhookのみサポートしています。将来のバージョンでSlackやWebhook APIの拡張を検討しています。

---

## 技術的な質問

### Q16: データベースのバックアップはどうすればいいですか？

**A:** 定期的に `plugins/Kuma-Report/reports.db` をコピーしてバックアップしてください。

```bash
# Linuxの場合（cronで自動化）
0 3 * * * cp /path/to/plugins/Kuma-Report/reports.db /backup/reports-$(date +\%Y\%m\%d).db
```

---

### Q17: プラグインをアップデートする方法は？

**A:**
1. サーバーを停止
2. `plugins/Kuma-Report/config.yml` と `reports.db` をバックアップ
3. 古いJARファイルを削除
4. 新しいJARファイルを配置
5. サーバーを起動

詳細は [インストールガイド](Installation.md#アップグレード) を参照してください。

---

### Q18: 他のプラグインと競合しますか？

**A:** 基本的に競合は発生しませんが、以下のプラグインとの併用が推奨されます:
- LuckPerms（権限管理）
- Geyser + Floodgate（Bedrock版対応）
- Vault（経済プラグイン連携、将来のバージョンで対応予定）

---

### Q19: パフォーマンスへの影響は？

**A:** Kuma-Reportは軽量に設計されており、パフォーマンスへの影響は最小限です。データベース操作は非同期で実行されるため、サーバーのメインスレッドには影響しません。

---

### Q20: MySQLに対応していますか？

**A:** 現在のバージョンではSQLiteのみサポートしています。MySQLサポートは将来のバージョンで追加予定です。

---

### Q21: APIはありますか？

**A:** はい。他のプラグインから Kuma-Report の機能を利用できます。詳細は [API ドキュメント](API-Documentation.md) を参照してください。

---

### Q22: ソースコードはどこで確認できますか？

**A:** GitHubで公開しています: https://github.com/humisan/Kuma-Report

---

### Q23: プラグインの開発に貢献できますか？

**A:** はい！プルリクエストを歓迎します。大きな変更を加える場合は、まずIssueで議論してください。

## トラブルシューティング

### Q24: プラグインが起動しません

**A:** 以下を確認してください:
- Javaバージョンが17以降か
- Minecraftバージョンが1.21以降か
- Paper/Spigot/Purpurを使用しているか

詳細は [トラブルシューティング](Troubleshooting.md) を参照してください。

---

### Q25: 通報が保存されません

**A:** 以下を確認してください:
1. データベースファイルの権限（Linux: 644）
2. サーバーログにエラーが出ていないか
3. ディスク容量が十分にあるか

---

### Q26: Discord通知が届きません

**A:** 以下を確認してください:
1. `config.yml` で `enabled: true` になっているか
2. Webhook URLが正しいか
3. サーバーからインターネットに接続できるか
4. Discordチャンネルの権限設定が正しいか

詳細は [Discord連携](Discord-Integration.md#トラブルシューティング) を参照してください。

---

### Q27: GUIが開きません

**A:** 以下を確認してください:
- Java版プレイヤーか（Bedrock版はFormが表示されます）
- インベントリが満杯ではないか
- 他のプラグインとの競合がないか

---

### Q28: 16進数カラーコードが表示されません

**A:** Minecraft 1.16未満では16進数カラーコードに対応していません。`config.yml` で `use-hex-colors: false` に設定してください。

---

### Q29: クールダウンが機能しません

**A:** `kumareport.bypass.cooldown` 権限を持っているプレイヤーは、クールダウンをバイパスできます。権限を確認してください。

```bash
lp user <プレイヤー名> permission check kumareport.bypass.cooldown
```

---

### Q30: 通知音が鳴りません

**A:** `config.yml` で以下を確認してください:

```yaml
notifications:
  play-sound: true  # これがtrueになっているか
  sound-volume: 1.0  # 0.0だと無音
```
